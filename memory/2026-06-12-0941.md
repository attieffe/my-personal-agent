assistant: Il blocco su script lunghi in DBeaver di solito ha 2-3 cause classiche:

**1. Timeout query (il più comune)**
Connection > Edit Connection > Driver properties (o Connection settings) → cerca `queryTimeout` o `socketTimeout` → mettilo a 0 (= illimitato)

Oppure: Window > Preferences > Editors > SQL Editor → "Query timeout" → 0

**2. Memoria heap Java esaurita**
DBeaver è Java. Se lo script produce molti risultati, crasha per OOM.
Apri `dbeaver.ini` (nella cartella di installazione) e aumenta:
```
-Xms256m
-Xmx2048m
```
(o anche 4096 se hai RAM)

**3. Connessione che cade (keep-alive)**
Per connessioni remote: Connection > Edit Connection > Connection settings > Keep-Alive interval → attivalo (es. ogni 30s)

**4. Auto-fetch dei risultati**
Se lo script è una serie di SELECT che ritornano milioni di righe, DBeaver cerca di caricarle tutte. Vai su SQL Editor preferences e disabilita "Auto-fetch results" o abbassa il fetch size.

---

**Alternativa pragmatica:** per script SQL molto lunghi e critici, usa direttamente il CLI del tuo DB (`psql`, `mysql`, `sqlplus`, ecc.) — DBeaver è fantastico per interattivo ma per batch pesanti il CLI nativo non si blocca mai.
