## Come funziona l'architettura

```
Software custom (POS) → API locale RT → Registratore Telematico Epson
                                               ↓ (automatico)
                                         Agenzia delle Entrate
```

Il RT è una **scatola nera fiscale omologata** — gestisce autonomamente l'invio dei corrispettivi all'AdE. Il tuo software non tocca quella parte.

---

## API Epson per RT

Epson espone un'interfaccia **HTTP REST/XML** locale sul RT (tipicamente `http://192.168.x.x/cgi-bin/fpmate.cgi`), chiamata **ePOS/FP Mate API**.

Puoi fare:
- Emissione scontrino fiscale
- Reso/annullo
- Stampa Z (chiusura giornaliera)
- Lettura totali e stato
- Pagamenti misti (contanti, carta, ecc.)

**SDK e doc ufficiali Epson:**
- `FP Series Communication Protocol` — documentazione XML
- `ePOS SDK for Android/iOS/JavaScript` — se vuoi anche mobile
- Disponibili sul sito Epson developers

## Vincoli da rispettare

| Cosa puoi fare | Cosa NON puoi fare |
|---|---|
| Software custom che comanda il RT | Bypassare la stampante fiscale |
| Integrare con gestionale, e-commerce, CRM | Modificare firmware/log fiscali |
| Multi-cassa, multi-pagamento | Emettere ricevute senza passare dal RT |
| Interfaccia custom (touch, web, app) | Interrompere trasmissione AdE |

---

## Stack consigliato

- **Backend:** Node.js o Python → HTTP verso RT
- **Frontend:** qualsiasi (React, app mobile, ecc.)
- **Comunicazione RT:** XML su HTTP (sincrono, semplice)

Il payload per uno scontrino è un XML tipo:

```xml
<printerFiscalReceipt>
  <beginFiscalReceipt />
  <printRecItem description="Prodotto A" quantity="1" unitPrice="10.00" department="1" />
  <printRecTotal method="0" operator="1" payment="10.00" description="Contanti" />
  <endFiscalReceipt cut="true" />
</printerFiscalReceipt>
```
