**Perché OpenClaw è lento — le cause reali**

Non è un problema di modello "stupido" o hardware insufficiente. È l'architettura che porta peso:

1. **Accumulo di contesto**: OpenClaw mantiene la cronologia completa della conversazione. Quello che inizia come 5k token diventa 150k token dopo 10-15 turni. Ogni query ri-manda tutto al modello.

2. **Thinking mode verboso**: il default fa un ragionamento chain-of-thought esteso prima di rispondere (~2.2s solo per quello). Per una chat Telegram è eccessivo.

3. **WebSocket throttle a 150ms**: deliberato, per non fare flooding della UI. Ma per uso messaggistica è controproducente — i token escono a pacchetti da 150ms invece che in streaming.

4. **Caricamento contesto da file**: ogni sessione rilegge MEMORY.md, file di progetto, workspace — utile, ma costa tempo in startup.
