Un `502 Bad Gateway` di solito vuol dire che il proxy/gateway ha ricevuto una risposta invalida o nessuna risposta dal server a monte. Le cause comuni sono:
- risorse esaurite sul backend: CPU, RAM, worker PHP-FPM, connessioni
- processo backend crashato o bloccato
- timeout tra proxy e app
- problemi di rete/DNS tra i componenti
- manutenzione o deploy andato male

Quindi: **risorse sì, ma non solo**. Se è un errore sporadico, spesso è overload o timeout transient. Se è continuo, guarderei subito:
- log di nginx/apache
- log PHP-FPM o dell’app
- CPU/RAM/IO del server
- stato del servizio backend
