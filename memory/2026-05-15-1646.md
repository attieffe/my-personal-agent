In pratica ci sono 3 livelli:

1. Adattamento del prompt
- cambia istruzioni, contesto, esempi
- è il livello più facile e sicuro

2. Memoria / routing / policy
- salva esiti, preferenze, errori
- usa quei dati per scegliere prompt, tool, branch o priorità
- qui “impara” davvero nel senso operativo

3. Auto-modifica del workflow
- il grafo n8n cambia solo se c’è un componente esterno che lo riscrive via API / file / deploy
- senza questo, n8n da solo non “evolve” la struttura dei nodi

Opportunità vere:
- memory layer per migliorare decisioni
- evaluator loop per correggere output sbagliati
- router che sceglie workflow diversi in base ai casi
- generatore di prompt dinamico
- self-healing: se un branch fallisce, prova alternative

Limite importante:
- l’autoapprendimento non è nativo come nei modelli ML
- è più corretto parlare di “sistema che aggiorna contesto, regole e routing”
