# Funzione `cm_redirect_users_by_role` - Gestione Redirect Utenti

**File**: `wp-content/themes/blancone/functions.php`
**Hook**: `admin_init` (blocca l'accesso all'admin WordPress e redirige)
**Condizione**: Si attiva solo se NON è una chiamata AJAX

---

## 📋 SCENARIO 1: Utente con UN SOLO RUOLO

| Ruolo | Destinazione Redirect |
|-------|----------------------|
| `centro_autorizzato` | `/dashboard/` |
| `paziente_centro` | `/area-riservata-ai-pazienti/` |
| `consulente_capoarea` | `/piattaforma-consulenti/` |
| `consulente_agente` | `/piattaforma-consulenti/` |
| `piattaforma_agenti_IDS` | `/piattaforma-consulenti/` |
| Altri ruoli | Nessun redirect (accesso admin normale) |

---

## 📋 SCENARIO 2: Utente con MULTIPLI RUOLI

**Logica**: Se l'utente NON è `administrator`, `ids_dental` o `promo_click`:

| Ruolo presente | Destinazione Redirect | Priorità |
|----------------|----------------------|----------|
| `paziente_centro` | `/area-riservata-ai-pazienti/` | 1 |
| `centro_autorizzato` | `/dashboard/` | 2 |
| `consulente_capoarea` / `consulente_agente` / `piattaforma_agenti_IDS` | `/piattaforma-consulenti/` | 3 |
| Nessuno dei precedenti | Homepage `/` | 4 |

---

## ✅ ECCEZIONI - Possono accedere all'admin WordPress

I seguenti ruoli **non** vengono rediretti e possono accedere all'admin:
- `administrator`
- `ids_dental`
- `promo_click`

---

## 📌 Note Tecniche

- **Priorità valutazione**: Nel caso di multipli ruoli, la priorità viene valutata dall'alto verso il basso (il primo ruolo trovato determina il redirect)
- **Exit**: Ogni redirect termina con `exit()` per bloccare ulteriore esecuzione
- **AJAX**: La funzione non si attiva durante chiamate AJAX per evitare conflitti
