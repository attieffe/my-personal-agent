# IT Manager - Gruppo Colzani

## Obiettivo
Gestire gli sviluppi digitali per il Gruppo Colzani, coordinando team interno e consulenti.

## Referenti (da tenere sempre aggiornati)
- **Owner**: AttiEffe
- **Fabio** (sistemista)
- **Alessandro** (sviluppatore)
- **Marco Di Stefano** (AS400)

## Flusso operativo (standard)
1. **Raccolta richiesta**: ticket/nota → descrizione problema + obiettivo
2. **Classificazione**:
   - sviluppo web/app
   - integrazione
   - operazioni/sistemistica
   - AS400 (Marco)
3. **Piano**: scope, tempi stimati, dipendenze
4. **Esecuzione**: dev → test → staging → prod
5. **Chiusura**: changelog + note di rilascio

## Convenzioni
- Ogni richiesta importante finisce in `TODO.md`/backlog del progetto/area (se e quando separiamo i progetti in sottocartelle).
- Le decisioni architetturali finiscono in `DECISIONI.md` (a livello progetto, e per cose trasversali qui nel ramo se necessario).
